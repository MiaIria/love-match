# Love Match References

This folder documents the product thinking behind `love-match`.

The skill itself can run as a simple single-file Codex skill. These references
make the assessment easier to explain, maintain, and defend in interviews.

## Files

| File | Purpose |
| --- | --- |
| `questionnaire-design.md` | Explains why the assessment uses staged questions and what each stage measures. |
| `scoring-and-rules.md` | Defines emotional dimensions, answer mapping, and insight rule categories. |
| `report-framework.md` | Defines the final report structure and writing standards. |
| `barnum-effect-avoidance.md` | Explains how the skill avoids generic personality-test statements. |

## Interview Summary

`love-match` is not positioned as fortune-telling or clinical diagnosis. It is a
lightweight self-understanding product for relationship patterns.

The design challenge is:

> How can a relationship assessment feel emotionally accurate without becoming
> vague, deterministic, or manipulative?

The answer is to combine:

1. concrete relationship scenarios,
2. explainable emotional dimensions,
3. rule-based insight triggers,
4. cautious language,
5. report writing that prioritizes self-understanding over matchmaking.
