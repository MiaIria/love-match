# Report Framework

## Report Goal

The report should first help the user understand themselves, then discuss
compatible partner types. It should not jump directly to "you should date this
kind of person".

Recommended weighting:

- 75% self-understanding,
- 25% partner compatibility.

## Recommended Structure

```markdown
# 恋爱匹配分析

## 你的性格画像

## 亲密关系中的核心模式

## 关系中的优势

## 可能遇到的挑战

## 匹配类型 1

## 匹配类型 2

## 匹配类型 3

## 相处建议

## 注意事项
```

## Writing Standards

Use language that is:

- warm,
- specific,
- non-judgmental,
- bounded,
- practical.

Avoid language that is:

- deterministic,
- mystical,
- manipulative,
- overly flattering,
- shame-inducing.

## Good Insight Pattern

```text
你可能不是不需要亲密，而是对进入亲密关系的速度比较敏感。
如果对方推进太快，你会先观察甚至后退；如果对方稳定、清楚、说到做到，
你反而会慢慢打开。
```

Why it works:

- It explains a behavior without blaming the user.
- It gives a condition under which the pattern changes.
- It is specific enough to be useful.

## Partner Recommendation Format

For each compatible type, include:

1. why this type fits,
2. what feels comfortable,
3. what may become difficult,
4. how to communicate,
5. what to observe in real interaction.

## Disclaimer

Always frame the report as a reflective tool:

```text
这份分析只适合作为自我理解和关系观察的参考，不是诊断，也不能保证一段关系的结果。
真实关系仍取决于沟通、边界、时机和双方意愿。
```

## Interview Talking Point

The report is designed to avoid cheap matchmaking. It starts from self-pattern
recognition because better partner choice depends on first understanding one's
own needs, defenses, and conflict style.
