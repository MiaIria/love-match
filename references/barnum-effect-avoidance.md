# Barnum Effect Avoidance

## Problem

Relationship assessments can easily produce statements that feel true because
they are vague enough to fit almost anyone.

Examples:

- You want to be loved but fear being hurt.
- You care about sincerity.
- You sometimes overthink.
- You need someone who understands you.

These statements may feel pleasant, but they are weak product output.

## Design Principle

Every meaningful claim should be anchored to evidence.

Evidence can be:

- a questionnaire answer,
- a repeated pattern across answers,
- a contradiction between dimensions,
- a user-provided story,
- a defined insight rule.

## Rewrite Pattern

Weak:

```text
你很重感情，但有时也会害怕受伤。
```

Stronger:

```text
你在表达喜欢时并不一定慢，但一旦关系进入不确定阶段，你更倾向于先观察而不是直接确认。
这说明你不是没有需求，而是需要稳定信号来降低风险感。
```

## Specificity Checklist

Before outputting an insight, check:

- Does this apply to too many people?
- Does it refer to a concrete behavior?
- Is there a reason this user, not everyone, gets this sentence?
- Can the sentence be traced to an answer or rule?
- Does it help the user make a better observation or decision?

## Language Rules

Prefer:

- "你可能倾向于..."
- "在不确定时..."
- "当对方推进太快时..."
- "这类模式可能表现为..."

Avoid:

- "你就是..."
- "你注定..."
- "你一定适合..."
- "你的灵魂伴侣是..."

## Interview Talking Point

I treated Barnum effect as a product quality risk. A result that "sounds right"
is not enough. The result needs to be specific, traceable, and useful for real
relationship decisions.
